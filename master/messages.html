Ongoing Messages¶

Audio¶

In order to stream the audio, the client sends audio with the audio to transcribe.

  • type (string): audio.
  • audio (string): one audio chunk (must match the encoding, the length and the sampleRateHertz indicated in connection setup).
  • timestamp (integer): timestamp of the beginning of audio.

Exemple:

{
    "type": "audio",
    "audio": "...",
    "timestamp": 1531587765
}

Transcript¶

We return transcripts to the client in transcript messages:

  • type (string): transcript.
  • blocId (string): id of the trancript.
  • transcript (string): transcript text.
  • isFinal (boolean): if true, then no more transcript associated to this blocId will be published.

Exemple:

{
    "type": "transcript",
    "blocId": "5b49d758ec2040000121aaf5",
    "transcript": "Hello, World.",
    "isFinal": true
}

If two transcripts are sent with the same blocId, only the last one should be considered. For each blocId, we will send a transcript with a isFinal field set to true.

Ping / pong¶

We expect to receive at least one message every 10s. If during the connection (captioner ready or not), you think you may have a large period without sending message, you must send ping message. We will send back a pong message.

  • type (string): ping or pong.

Exemple:

{
    "type": "ping",
}
