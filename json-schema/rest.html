REST API¶

Basics¶

  • you must use JSON-encoded bodies for POST request, and therefore set the appropriate header Content-type: application/json). JSON body should be a hash of attribute key/value pairs (no array, string, number or boolean).
  • You must transmit a valid API token as a bearer token in the Authorization HTTP header.

Exemple:

POST /conversations HTTP/1.1
Content-type: application/json
Authorization: Bearer xxxxxxxxxxxxxxxxx

{"key": "value"}

Conversations¶

POST /conversations¶

In order to open start a conversation, you need to perform a POST call on /conversations. You will get in return a conversation object, with a single-use websocket url for audio / transcripts transfer. Single-use websocket url are only valid 30s.

Rrequest:

  • encoding (string): Encoding of audio data which will be sent. Valid value is: LINEAR16.
  • sampleRateHertz (integer): Sample rate in Hertz of the audio data which will be sent. Valid values are: 8000, 16000.
  • chunkLengthMs (integer): Duration in ms of the chunks of audio data which will be sent.
  • dev (boolean): optional - If true, then no real captioner will be triggered. A ready event will be sent in the websocket between 2 and 5 seconds after the connection initialization, and the following transcripts will be ASR only.
  • language (string): optional - Language of the conversation (french by default). Valid values are: fr-FR, en-US.

Example:

{
  "encoding": "LINEAR16",
  "sampleRateHertz": 8000,
  "chunkLengthMs": 100
}

Response:

  • url (string): optional - the url to connect to.
  • error (object): optional - An error object (see Errors).
  • id (string): optional - Id of the conversation. May be used to establish a second websocket connection (future) or to restart a connection if the first one is closed unexpectedly (future).

Typical success response (201):

{
  "url": "wss://scribe.ava.me/ws/5b49d758ec2040000121aaf?5ee3b6618774d422e3b",
  "id": "5b49d758ec2040000121aaf5"
}

Typical error response (400):

{
  "error": {
    "name": "VALIDATION_ERROR",
    "fields": {
      "encoding": "Invalid value"
    }
  }
}
